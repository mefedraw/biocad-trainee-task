# Тестовое задание BIOCAD 2026

## Описание проекта

RAG-система предназначена для помощи исследователям в поиске и анализе информации о потенциальных мишенях для разработки лекарств от болезни Альцгеймера.

Используя векторный поиск и большие языковые модели, система извлекает релевантную информацию из научных статей и генерирует структурированные ответы с цитированием источников. Если вопрос не является релевантным, система прямо об этом заявляет, **не предоставляя источников**.

## Архитектура системы

### Компоненты системы

1. **Text Extractor** — извлечение текста из PDF-документов  
2. **Chunker** — разбиение документов на семантически связанные чанки  
3. **Embedder** — создание векторных представлений текста  
4. **Vector Store** — хранение и поиск по векторным представлениям  
5. **Retriever** — поиск релевантных документов по запросу  
6. **Reranker** — переранжирование результатов для повышения точности  
7. **Generator** — генерация ответов на основе найденных документов  

## Процесс работы системы

### Этап 1: Подготовка данных

1. **Извлечение текста**
   - использование GROBID для структурированного извлечения (приоритет);
   - сохранение с метаданными.

2. **Чанкинг**
   - разбиение документов на чанки с учетом секций (abstract, introduction, conclusion);
   - сохранение контекста секций в метаданных;
   - перекрытие чанков для сохранения контекста;
   - сохранение.

3. **Создание эмбеддингов**
   - генерация векторных представлений с помощью PubMedBERT;
   - индексация в ChromaDB с использованием HNSW для быстрого поиска.

### Этап 2: Обработка запросов

1. **Retrieval**
   - кодирование пользовательского запроса в сжатое векторное представление;
   - поиск топ-K наиболее похожих чанков по косинусному расстоянию.

2. **Reranking**
   - использование cross-encoder модели `ms-marco-MiniLM-L-6-v2`;
   - более точная оценка релевантности запрос–документ;
   - отбор топ-8 наиболее релевантных чанков.

3. **Generation**
   - формирование контекста из найденных чанков с метаданными;
   - генерация ответа через Groq API моделью `openai/gpt-oss-120b`;
   - постобработка для исправления формата цитирования;
   - строгие правила цитирования.

**Запуск метрик:**
```bash
python scripts/evaluate.py --with-generation --limit 5
```

## Часть 4. Вопросы

### На какие модальности данных можно расширить решение?

решение можно расширить на структурированные данные по типу таблиц, на изображения тоже можно, и на комбинацию изображений и таблиц

### Как это можно сделать?

Для структурированных данных (таблиц) можно использовать модель TabNet для получения эмбеддингов строк таблицы. TabNet создает векторные представления которые затем можно индексировать в векторной базе данных аналогично текстовым чанкам

Для изображений можно использовать ViT для извлечения признаков из изображений или CLIP для мультимодальных эмбеддингов (но решение получится тяжелым думаю) 


### Какие модели и почему вы выбрали для решения

**PubMedBERT**:
Она специализирована на биомедицинских текстах и поэтому лучше понимает медицинскую терминологию и контекст

**Cross-Encoder**
CE оптимизирован для ранжирования и поэтому можно получить более точную оценку релевантности чем просто косинусное сходство

**Groq `openai/gpt-oss-120b`** :
На самом деле я ее выбрал потому что она является бесплатной и соответствует требованиям проекта, возможно для прода понадобится что то другое, хотя как по мне этой модели хватает с головой. Можно также конкретизировать промпт чтобы еще больше стандартизировать вывод, чтобы исследователям было удобно читать ответ и пользоваться системой.
